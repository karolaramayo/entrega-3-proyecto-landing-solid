SRP/ Single Responsibility Principle
Principio de responsabilidad única

La noción de que un objeto sólo
debería tener una única
responsabilidad.

En ingeniería de software establece que cada módulo o

clase

Una clase es una plantilla para la creación de objetos de datos según un modelo predefinido.

debe tener responsabilidad sobre una sola parte de la funcionalidad proporcionada por el software y esta, debe estar encapsulada en su totalidad por la clase. Todos sus servicios deben estar estrechamente alineados con esa responsabilidad: Robert C. Martin expresa el principio de la siguiente forma: En programación orientada a objetos, cada clase debería tener una única responsabilidad, y esta debe estar contenida únicamente en dicha clase, así: 1. Una clase debería tener solo una razón para cambiar 2. Cada responsabilidad es el eje del cambio 3. Para contener la propagación del cambio, debemos separar las responsabilidades 4. Si una clase asume más de una responsabilidad, será más sensible al cambio. 5. Si una clase asume más de una responsabilidad, las responsabilidades se acoplan.
